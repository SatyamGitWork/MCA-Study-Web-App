1. NPM: Node Package Manager

NPM (Node Package Manager) is the default package manager for Node.js and the largest ecosystem of open-source libraries (called "packages") in the world. Its primary job is to manage your project's dependenciesâ€”the external code your project relies on.

Main Components of NPM

The CLI (Command Line Interface)

The `npm` tool you use in your terminal to install packages (`npm install express`), run scripts (`npm run dev`), and initialize projects (`npm init`).

The NPM Registry

A massive public online database where all open-source packages are stored. `npm install` searches this registry.

The `package.json` File

The manifest file for your project. It lists metadata, dependencies, dev dependencies, and scripts.

The `node_modules` Folder

The local folder where NPM downloads and stores the actual code for all your project's dependencies.

2. `fs.writeFile()`

This method from the `fs` module asynchronously writes data to a file. It creates the file if it doesn't exist, and **overwrites** it if it does. 

// Import the 'fs' module
const fs = require('fs');

// Define the content to write
const content = 'Hello, Node.js! This file was created by fs.writeFile().';

// Define the file path
const filePath = 'welcome.txt';

// Asynchronously write data to the file
fs.writeFile(filePath, content, 'utf8', (err) => {
  if (err) {
    console.error('An error occurred while writing the file:', err);
    return;
  }
  
  console.log('File has been created and saved successfully!');
});

3. `fs.readFile()`

This method asynchronously reads the entire contents of a file into memory. It's crucial to specify an encoding (like `'utf8'`) to get a string, otherwise you'll get a raw buffer. 

// Import the 'fs' module
const fs = require('fs');

// Define the file path to read
const filePath = 'welcome.txt';

// Asynchronously read the file
fs.readFile(filePath, 'utf8', (err, data) => {
  if (err) {
    // Handle errors (e.g., file not found)
    console.error('An error occurred while reading the file:', err);
    return;
  }
  
  // 'data' contains the file content as a string
  console.log('File content:');
  console.log(data);
});

4. Delete a File (`fs.unlink()`)

To delete a file asynchronously, you use the `fs.unlink()` method. 

// Import the 'fs' module
const fs = require('fs');

// Define the file path to delete
const filePath = 'welcome.txt';

// Asynchronously delete the file
fs.unlink(filePath, (err) => {
  if (err) {
    console.error('An error occurred while deleting the file:', err);
    return;
  }
  
  console.log('File has been deleted successfully!');
});

5. The Operating System (`os`) Module

The `os` module provides operating system-related utility methods and properties. You can access it with `const os = require('os');`.

`os.platform()`

Type: Method. Returns a string identifying the OS platform (e.g., `'win32'`, `'linux'`, `'darwin'`). Useful for writing system-specific code.

const os = require('os');
console.log('Platform:', os.platform());

`os.totalmem()`

Type: Method. Returns the total amount of system memory (RAM) in bytes as an integer. Useful for system monitoring.

const os = require('os');
const totalMemoryGB = os.totalmem() / (1024 * 1024 * 1024);
console.log(`Total Memory: ${totalMemoryGB.toFixed(2)} GB`);

`os.cpus()`

Type: Method. Returns an array of objects, where each object contains information about a logical CPU core on the system.

const os = require('os');
const cpus = os.cpus();
console.log(`Total Cores: ${cpus.length}`);
console.log('First Core Model:', cpus[0].model);

6. Explain `app.js` File

While `app.js` isn't a special file to Node.js itself, it has become a strong community convention for naming the main entry point of a web application, especially when using the Express framework.

Purpose of `app.js`

In a typical Express server, `app.js` is responsible for:

  • Importing and initializing the Express framework.
  • Setting up server configurations and middleware (like `express.json()`).
  • Defining the application's routes (URLs) and their handlers.
  • Starting the server to listen for connections on a specific port.

Example `app.js` (using Express)

To run this, you must first run `npm init -y` and `npm install express`.

// 1. Import Dependencies
const express = require('express');
const os = require('os');

// 2. Initialize the Express application
const app = express();

// 3. Define a port
const PORT = 3000;

// 4. Configure Middleware (e.g., to parse JSON bodies)
app.use(express.json());

// 5. Define Routes
// A simple 'GET' route for the homepage
app.get('/', (req, res) => {
  res.send('Welcome to the homepage!');
});

// A 'GET' route to get system info
app.get('/api/sysinfo', (req, res) => {
  const info = {
    platform: os.platform(),
    totalMemory: os.totalmem(),
  };
  res.json(info);
});

// 6. Start the Server
app.listen(PORT, () => {
  console.log(`Server is running on http://localhost:${PORT}`);
});

To run this file, you would save it as `app.js` and execute `node app.js` in your terminal.

7. More File System Methods

Here are three other common `fs` module methods.

`fs.mkdir()` - Create a Directory

Asynchronously creates a new directory. Using `{ recursive: true }` prevents errors if the directory exists and creates parent directories if needed.

const fs = require('fs');
const path = require('path');

const dirPath = path.join(__dirname, 'my-new-directory');

fs.mkdir(dirPath, { recursive: true }, (err) => {
  if (err) {
    console.error('Error creating directory:', err);
    return;
  }
  console.log('Directory created successfully!');
});

`fs.readdir()` - Read a Directory

Asynchronously reads the contents of a directory, returning an array of filenames.

const fs = require('fs');

// Read the current directory
fs.readdir(__dirname, (err, files) => {
  if (err) {
    console.error('Error reading directory:', err);
    return;
  }
  
  console.log('Files in this directory:', files);
});

`fs.rename()` - Rename a File or Directory

Asynchronously renames a file or directory.

const fs = require('fs');

const oldPath = 'hello.txt'; // A file you created earlier
const newPath = 'greetings.txt';

// First, create the file to be renamed
fs.writeFile(oldPath, 'Hello!', (err) => {
  if (err) return console.log('Error creating file:', err);
  
  console.log('File created. Now renaming...');
  
  // Now, rename it
  fs.rename(oldPath, newPath, (err) => {
    if (err) {
      console.error('Error renaming file:', err);
      return;
    }
    console.log('File renamed successfully to greetings.txt');
  });
});

8. How to use Async/Await

`Async/Await` is modern syntax that makes working with asynchronous Promises much cleaner and easier to read.

  • `async`: Keyword to declare a function that will handle asynchronous operations. An `async` function always returns a Promise.
  • `await`: Keyword used *inside* an `async` function to pause its execution until the Promise it's "awaiting" is resolved or rejected.
  • Error Handling: You use standard `try...catch` blocks.

To use async/await with the `fs` module, you must use the `fs.promises` API:

// Import the 'promises' version of the fs module
const fs = require('fs').promises;

const filePath = 'welcome.txt'; // Assuming this file exists

// Define an async function
async function readFileAsync() {
  try {
    // 'await' pauses execution until the file is read
    const data = await fs.readFile(filePath, 'utf8');
    
    // This line only runs after the 'await' is complete
    console.log('File content:');
    console.log(data);
    
  } catch (err) {
    // Any error (e.g., file not found) is caught here
    console.error('An error occurred:', err);
  }
}

// Call the async function
readFileAsync();

10. Error Handling Patterns

Error handling is critical in Node.js. Unhandled errors will crash your application. Here are the primary patterns.

Pattern 1: Error-First Callbacks (Classic)

The callback function's *first* argument is *always* reserved for an `error` object. You must always check for `err` first.

const fs = require('fs');
fs.readFile('non-existent-file.txt', (err, data) => {
  // 1. ALWAYS check for the error first
  if (err) {
    console.error('Callback Error:', err.message);
    return; // Stop execution
  }
  
  // 2. If no error, 'err' is null, and you can use 'data'
  console.log('File data:', data);
});

Pattern 2: Promises (`.catch()`)

Promise-based functions return a Promise. You handle success with `.then()` and errors with `.catch()`.

const fs = require('fs').promises; // Promise-based fs

fs.readFile('non-existent-file.txt', 'utf8')
  .then(data => {
    // This runs on success
    console.log('File data:', data);
  })
  .catch(err => {
    // This runs if ANY error occurs in the chain
    console.error('Promise Error:', err.message);
  });

Pattern 3: Async/Await (`try...catch`)

The cleanest way. You wrap your `await` calls in a `try...catch` block.

const fs = require('fs').promises;
async function readMyFile() {
  try {
    const data = await fs.readFile('non-existent-file.txt', 'utf8');
    console.log('File data:', data);
  } catch (err) {
    console.error('Async/Await Error:', err.message);
  }
}
readMyFile();

Pattern 4: Uncaught Exceptions (Last Resort)

This listener catches errors that were not handled anywhere else. It's a last resort for logging before gracefully shutting down. Do not use this to continue running.

process.on('uncaughtException', (err) => {
  console.error('!!! UNCAUGHT EXCEPTION !!!', err.message);
  console.error('Stack:', err.stack);
  
  // Best practice: Log the error and exit gracefully
  process.exit(1);
});
