Soft Computing

Unit IV Study Guide

Definition

A computer system designed to emulate the decision-making ability of a human expert. It solves complex problems in a narrow domain.

Key Components

  • Knowledge Base (KB) Repository of facts and rules.
  • Inference Engine The "brain" that applies rules to facts to derive conclusions.
  • User Interface Communication channel for input/output.
  • Explanation Module Explains "Why" and "How" a conclusion was reached.

Famous Examples

MYCIN (Bacteria) DENDRAL (Chemistry) PROSPECTOR (Minerals) XCON/R1 (Computer Config)
Match Resolve Act

Forward Chaining

Data-Driven: Starts with facts, fires rules to reach a goal. Used for planning/monitoring.

Backward Chaining

Goal-Driven: Starts with hypothesis, works backward to find facts. Used for diagnosis.

Frame-Based Systems

Organizes knowledge into "Frames" (like objects/classes) with slots for attributes. Supports inheritance.

Blackboard Systems

Collaborative architecture where independent "knowledge sources" write to a shared global memory (Blackboard).

Case-Based Reasoning (CBR)

Solves new problems by retrieving/adapting solutions from similar past cases. Mimics analogical reasoning.

Expert System Shells

Frameworks without a KB (e.g., EMYCIN, CLIPS). Provides the skeleton for rapid development.

Deals with approximate reasoning (vagueness) rather than exact binary logic.

Crisp Set 0 or 1 (Binary) "Tall" or "Not Tall"
Fuzzy Set Degree [0, 1] "0.8 Tall"
Linguistic Variables: Words like "High", "Medium", "Low" used as values.

Artificial Neural Networks (ANN)

Computational models inspired by the human brain.

  • Layers: Input, Hidden (Deep Learning), Output.
  • Backpropagation: Algorithm to adjust weights by propagating error backward.

Genetic Algorithms (GA)

Optimization technique inspired by Darwinian evolution.
