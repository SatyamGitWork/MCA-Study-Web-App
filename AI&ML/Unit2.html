AI Strategies

Unit II Study Guide

A control strategy is the logic used to select rules and track those already applied. In production systems, we often lack immediate knowledge to pick the best rule, necessitating search processes.

Definition

Irrevocable Strategy
  • An applicable rule is selected and applied irrevocably.
  • No provision to reconsider later (no "undo").

Definition

Tentative Strategy

Allows returning later if a solution isn't found.

  • Backtracking: Revert to a previous point if difficulty is encountered.
  • Graph Search: Tracks multiple rule sequences simultaneously.

Finding a Path

Focuses on finding any satisfactory path to a solution. Examples: DFS, BFS.

Finding the Best Path

Focuses on finding the optimal path (least cost). Examples: A*, Branch & Bound.

Game Playing

Involves adversaries (opponents). Examples: Minimax.

A. Depth-First Search (DFS)

Dives deep into the tree, picking one alternative at each level.

Algorithm Logic

1. Form queue with root.
2. While queue not empty:
   - If first is goal, Stop.
   - Remove first, add children to FRONT (Stack).

B. Hill Climbing

DFS attached to a "Quality Measurement". Uses heuristics to order choices.

Algorithm Logic

Similar to DFS, but sorts children by estimated remaining distance before adding to the front.

Problem: Local Maximum

A state better than neighbors but not the global peak.

Problem: Plateau

A flat area where neighbors have the same value.

C. Breadth-First Search (BFS)

Explores level-by-level.

Algorithm Logic

Remove first element, add children to the BACK of the queue.

A. British Museum Search

Finds ALL possible paths and selects the best. Only feasible for tiny trees.

B. Branch and Bound

Handles multiple partial paths. The shortest path is expanded.

Logic

Sort the queue by accumulated cost. First path to goal is optimal.

C. A* Search (A-Star) Important

Combines Branch & Bound with heuristic estimate.

f(n) = g(n) + h(n)

A. Minimax Search

For two-player adversarial games.

MAX Maximizes score
MIN Minimizes score

B. Alpha-Beta Pruning

Optimization for Minimax.

Pruning Logic

  • Alpha (α): Best value MAX can guarantee.
  • Beta (β): Best value MIN can guarantee.
  • Stop if α ≥ β
